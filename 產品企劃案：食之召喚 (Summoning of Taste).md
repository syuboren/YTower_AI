# **產品企劃案：食之召喚 (Summoning of Taste)**

## **—— 冰箱融合戰記：將清冰箱轉化為一場史詩級的召喚儀式**

版本：1.0  
日期：2025-12-14  
提案人：AI 協作專案組

## **1\. 專案摘要 (Executive Summary)**

**「食之召喚」** 是一款結合 **生成式 AI (GenAI)** 與 **卡牌戰鬥遊戲 (TCG)** 機制的創新食譜推薦應用。

痛點分析：  
現代人面臨「不知道煮什麼」、「冰箱剩材浪費」以及「傳統食譜網站枯燥乏味」的三大痛點。  
解決方案：  
本產品將枯燥的備餐過程遊戲化（Gamification）。使用者不再是「查食譜」，而是扮演「鍊金術師」，將冰箱中的食材視為「素材卡牌」，透過 AI 引擎的「融合儀式」，召喚出高星級的「美味料理」。這不僅解決了剩食問題，更創造了高黏著度的使用者體驗。

## **2\. 核心概念與世界觀 (Core Concept)**

### **2.1 遊戲化隱喻 (Metaphors)**

| 現實行為 | 遊戲化包裝 | 視覺/互動呈現 |
| :---- | :---- | :---- |
| **輸入食材** | **具現化 (Materialize)** | 輸入文字後，經過數據掃描與粒子重組，生成一張擁有屬性與數值的實體卡牌。 |
| **AI 搜尋運算** | **融合儀式 (Fusion)** | 將卡牌拖入魔法陣，啟動螺旋光效與數據流，模擬鍊金術過程。 |
| **食譜推薦** | **召喚怪獸 (Summon)** | 產出 3-5 張食譜卡，卡片具有攻擊力（美味）、防禦力（健康）與星級。 |
| **烹飪難度** | **星級 (Level)** | 星星越多代表製作難度越高，但也越稀有。 |
| **食材屬性** | **元素屬性 (Attribute)** | 肉類(炎)、海鮮(水)、蔬菜(地/風)、調味(光/暗)。 |

### **2.2 使用者旅程 (User Journey)**

1. **啟動終端**：使用者打開 App，進入充滿科技魔法感的暗黑介面。  
2. **盤點手牌**：輸入冰箱現有食材（如：牛肉、洋蔥），系統即時生成素材卡。  
3. **策略組牌**：使用者從手牌中挑選 2-3 張關鍵素材放入「融合陣」。  
4. **發動融合**：點擊按鈕，欣賞 AI 運算過場動畫。  
5. **揭曉結果**：翻開覆蓋的食譜卡，查看 AI 推薦的料理數值與攻略。  
6. **實作攻略**：點擊卡片進入詳細烹飪步驟（戰鬥教學）。

## **3\. 功能規格 (Feature Specifications)**

### **3.1 核心功能 (MVP 階段)**

* **AI 物質具現化系統**：  
  * 支援文字輸入與語音輸入。  
  * 自動將食材關鍵字對應至圖庫，若無對應圖片則生成屬性圖騰。  
  * 自動賦予食材屬性（例如：辣椒自動歸類為「炎」屬性）。  
* **RAG 融合引擎**：  
  * 基於 RAG (Retrieval-Augmented Generation) 技術。  
  * 根據投入的素材卡，從向量資料庫中檢索匹配度最高的食譜。  
  * 容錯機制：若素材不足，AI 自動補完建議（「提示：再加入一張『馬鈴薯』可召喚更強怪獸」）。  
* **動態卡牌展示**：  
  * 高互動性的卡牌 UI（懸停浮動、點擊翻轉）。  
  * 視覺化的數據雷達圖（ATK/DEF/SPD）。

### **3.2 進階功能 (Phase 2\)**

* **圖鑑系統 (The Grimoire)**：記錄使用者曾經「召喚」成功的所有料理，滿足收集癖。  
* **每日任務 (Daily Quests)**：例如「今日屬性加成：水屬性料理」，完成獲得虛擬獎勵。  
* **陷阱卡預警**：若輸入相剋食材（如：螃蟹 \+ 柿子），系統彈出紅色警報卡片。

## **4\. UI/UX 設計策略**

* **視覺風格 (Visual Style)**：  
  * **Dark Mode Only**：深邃宇宙黑背景，搭配霓虹光（黃/紫/青）點綴，強調「魔法」與「科技」的結合。  
  * **高質感影像**：食譜圖片必須是高解析度、令人垂涎的「Food Porn」等級。  
  * **微互動 (Micro-interactions)**：輸入時的掃描線、卡片生成時的上浮動畫、融合時的震動回饋。  
* **互動邏輯**：  
  * **Drag & Drop**：直覺地將食材拖入鍋子（魔法陣）。  
  * **Haptic Feedback**：手機端在融合成功時給予震動回饋。

## **5\. 技術架構 (Technical Architecture)**

* **前端 (Frontend)**：  
  * **Framework**: Angular (Standalone Components) / React.  
  * **Styling**: Tailwind CSS (快速刻畫複雜 UI).  
  * **Animations**: CSS Keyframes \+ Framer Motion (React) / Angular Animations.  
* **後端 (Backend)**：  
  * **Server**: Node.js / Python (FastAPI).  
  * **AI/LLM**: OpenAI GPT-4o 或 Gemini Pro (負責理解食材關聯與生成食譜描述).  
  * **Database**:  
    * Vector DB (Pinecone / Milvus): 儲存食譜的向量特徵，用於 RAG 快速檢索。  
    * Relational DB (PostgreSQL): 儲存使用者圖鑑與帳號資料。

## **6\. 商業模式 (Business Model)**

1. **導購與電商結合 (缺牌補給)**：  
   * 當融合出的食譜缺少某一項關鍵食材（例如缺「鮮奶油」），顯示為灰階的「未裝備卡片」。  
   * 點擊卡片直接連結至合作電商（如：全聯、家樂福、楊桃美食網）進行購買。  
2. **品牌聯名卡 (Brand Partnership)**：  
   * 與醬料品牌合作，推出「稀有裝備卡」（例如：龜甲萬醬油）。使用該卡片融合，ATK（美味度）加成 20%。  
3. **訂閱制 (Premium)**：  
   * 解鎖「限定卡背」。  
   * 優先使用更高階的 AI 模型（生成更精準、更具創意的私房菜）。

## **7\. 開發時程規劃 (Roadmap)**

* **Month 1: 原型驗證 (POC)**  
  * 完成前端卡牌互動效果（目前進度）。  
  * 串接基礎的 OpenAI API 進行文字轉食譜測試。  
* **Month 2: 資料庫建置**  
  * 建立 500 道基礎食譜的向量資料庫。  
  * 完善「食材 \-\> 屬性」的對照表。  
* **Month 3: MVP 開發**  
  * 完成手機版 RWD 介面。  
  * 實作使用者帳號與收藏功能。  
* **Month 4: 測試與發布**  
  * 封閉測試 (Alpha Test)。  
  * 正式上線 (Launch)。
